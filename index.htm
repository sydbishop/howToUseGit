Setting Up the Environment

Install Git and VS Code

Before you can start using GitHub with VS Code, make sure you have Git and Visual Studio Code installed on your system.

  • Git: Download and install Git from here.
  • VS Code: Download and install Visual Studio Code from here.
Configure Git within VS Code

Configure your Git account with your GitHub. To do this, open Command Prompt and type:

git config --global user.name “YourUserNameOnGithub”

git config --global user.email “YourEmail”

Here, replace YourUserNameOnGithub with your GitHub-username, and YourEmail with the email-id used while creating an account on GitHub.

This process ensures that your Git commits will be associated with the correct user.

Creating and Cloning a GitHub Repository

Create a New Repository on GitHub
  1. Go to GitHub and log in to your account.
  2. Click on the '+' icon in the top right corner and select "New repository."
  3. Fill in the repository details, add a README if needed, and click "Create repository."
Clone the Repository using VS Code
  1. Copy the repository URL from GitHub.
  2. In VS Code, open the command palette (Ctrl+Shift+P), type "Git: Clone," and paste the repository URL.
  3. Choose a local folder, and VS Code will clone the repository for you.

Basic Git Operations in VS Code

Status

Use the Source Control view in VS Code to check the status of your changes.

Add

Stage changes by clicking the "+" button next to the files you want to include.

Commit

Enter a commit message and click the checkmark to commit your changes.

Push/Pull

Use the sync icon in the bottom-right corner to push or pull changes.

Branching with VS Code

Create and Switch Between Branches
  1. Open the Source Control view, click on the branch name at the bottom, and select "Create Branch."
  2. Enter a branch name and press Enter.
  3. To switch branches, click on the branch name and select the desired branch.
Understand the Concept of Branching

Branching allows you to work on features or bug fixes in isolation without affecting the main codebase. It enables parallel development and easy collaboration.

Working with Remote Repositories

Add and Manage Remote Repositories
  1. Open the Source Control view, click on the three dots, and select "Remote."
  2. Click on the '+' icon to add a remote repository.
Push to and Pull from Remotes
  • Use the sync icon to push changes to the remote repository.
  • To pull changes, click on the sync icon and choose "Pull from..."

Pull Requests and Code Reviews within VS Code

Create Pull Requests
  1. On GitHub, navigate to your repository and click on the "Pull requests" tab.
  2. Click "New pull request," select the base and compare branches, and click "Create pull request."
Review and Discuss Pull Requests
  1. In VS Code, open the "Pull Requests" view.
  2. Select the pull request you want to review.
  3. Comment on the changes and discuss with your collaborators.
